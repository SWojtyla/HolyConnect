## Key Features

### 1. Separate Flows Menu Item
- **Location**: Left sidebar navigation menu
- **Icon**: ğŸŒ³ Account Tree
- **Position**: Between Environments and Git
- **Route**: `/flows`

### 2. Dedicated Flows Page
- **Card-based layout**: Easy to scan multiple flows
- **Flow metadata**: Name, description, environment, collection, step count
- **Quick actions**: View, Edit, Execute, Delete
- **Empty state**: Helpful message with create action

### 3. Flow Management
- **Create**: Add new flows with multiple steps
- **Edit**: Modify existing flows (environment locked)
- **Delete**: Remove flows with confirmation
- **Execute**: Run flows and see results

### 4. Step Management
- **Add/Remove**: Easily manage flow steps
- **Reorder**: Move steps up/down
- **Configure**: Enable/disable, error handling, delays
- **Validation**: Prevents invalid step creation

### 5. Data Persistence
- **File-based storage**: Each flow saved as separate JSON file
- **Location**: `{StoragePath}/flows/{FlowId}.json`
- **Performance**: Multi-file repository for better scaling

## User Workflows

### Creating a Flow
1. Click "Flows" in the navigation menu
2. Click "Create Flow" button
3. Fill in flow name and description
4. Select environment (and optionally collection)
5. Add steps by selecting requests
6. Configure each step (enable/disable, error handling, delay)
7. Reorder steps as needed
8. Click "Create Flow"

### Editing a Flow
1. Navigate to Flows page
2. Click â‹® menu on flow card
3. Select "Edit"
4. Modify flow details and steps
5. Click "Save Flow"

### Executing a Flow
1. Navigate to Flows page
2. Click "Execute" button on flow card
3. View real-time execution progress
4. Review step results and responses

### Deleting a Flow
1. Navigate to Flows page
2. Click â‹® menu on flow card
3. Select "Delete"
4. Confirm deletion in dialog

### Components
- `Flows.razor` - Main flows listing page
- `FlowCreate.razor` - Flow creation form
- `FlowEdit.razor` - Flow editing form
- `FlowExecute.razor` - Flow execution and results
- `NavMenu.razor` - Updated with Flows link

### Services
- `IFlowService` - Flow management interface
- `FlowService` - Implementation with full CRUD operations
- `IRepository<Flow>` - Persistence layer
- `MultiFileRepository<Flow>` - File-based storage

### Architecture
Follows clean architecture principles:
- **Domain**: Flow, FlowStep, FlowExecutionResult entities
- **Application**: FlowService business logic
- **Infrastructure**: MultiFileRepository persistence
- **Presentation**: Blazor MAUI pages and components
